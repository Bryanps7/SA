Fundamentos de Redes

A página de Redes é dedicada ao estudo e à aplicação de tecnologias que conectam dispositivos, permitindo a troca de informações de forma eficiente e segura. Aqui você encontrará conteúdos sobre a configuração de equipamentos de rede, como switches e roteadores, os principais protocolos utilizados (DNS, HTTP, FTP, SMTP, entre outros), e conceitos fundamentais como o modelo OSI e a pilha TCP/IP

Dispositivos para um Rede Interna

Uma rede interna típica com um servidor web geralmente inclui:

Roteador

Interliga diferentes redes e direciona pacotes de dados entre elas, garantindo comunicação externa.

Switches

Conecta dispositivos em uma rede local, permitindo a troca de dados entre eles.

Servidor

Dispositivo que oferece serviços, como páginas web, resolução de nomes ou distribuição de IPs, para outros dispositivos na rede.

Host(PC ou Laptop)

Computadores ou dispositivos que acessam os serviços disponíveis na rede.

Cabo de Conexão

Proporcionam a conexão física entre os dispositivos para transmissão de dados.

Configurações dos Servidores

Nas Configurações do Servidor da Rede Interna, o mais importante é:

DNS

Configurações incluem entrada de registros para mapear nomes de domínio a endereços IP.

Exemplo:

sa-bryan.netlify.app/
HTTP

Necessário especificar o diretório raiz onde os arquivos HTML e outros conteúdos estão armazenados.

DHCP

Configuração de um escopo com intervalo de endereços IP, máscara de sub-rede, gateway padrão e DNS.

Conceito de Servidor

Um servidor é um sistema que fornece recursos, serviços ou dados para outros dispositivos, chamados clientes, em uma rede. Ele pode ser hardware ou software, projetado para alta performance, segurança e disponibilidade. Servidores são usados para funções como hospedar sites, gerenciar arquivos, bancos de dados e e-mails, entre outros, respondendo às solicitações feitas pelos clientes.

Vídeo da Crimpagem da Empresa

Para realizar a crimpagem, utilizamos um alicate de crimpagem específico e conectores RJ45 de alta qualidade. Cortamos o cabo de rede no tamanho desejado, organizamos os fios na sequência correta que é o padrão T568A e fixamos os conectores com precisão. Garantimos conexões seguras e funcionais para um excelente desempenho da rede.

Cabo de Rede - Pinagem

Pinagem padrão de um cabo de rede (RJ45) segue:

Padrão T568A

Verde/branco, verde, laranja/branco, azul, azul/branco, laranja, marrom/branco, marrom.

Padrão T568B

Laranja/branco, laranja, verde/branco, azul, azul/branco, verde, marrom/branco, marrom.

Conceito do modelo OSI

1. Física

Transmissão de bits pelo meio físico (cabos, sinais).

2. Enlace

Transferência confiável dentro da mesma rede (Ethernet, Wi-Fi).

3. Rede

Roteamento de pacotes entre redes (IP).

4. Transporte

Entrega fim a fim de dados (TCP, UDP).

5. Sessão

Gerencia conexões entre dispositivos.

6. Apresentação

Converte e formata os dados (criptografia, compressão).

7. Aplicação

Interage com o usuário (HTTP, FTP, DNS).

O modelo OSI (Open Systems Interconnection) é um modelo em 7 camadas que padroniza a comunicação em redes.

Conceito do TCP/IP

1. Aplicação

Oferece serviços para aplicativos (HTTP, DNS, SMTP).

2. Transporte

Garante a entrega dos dados:

  • TCP: Entrega confiável, com verificação de erros.
  • UDP: Entrega rápida, mas menos confiável.

3. Internet

Faz o roteamento e o endereçamento dos pacotes (IP, ARP).

4. Acesso à Rede

Gerencia a conexão física e o envio de pacotes na rede local (Ethernet, Wi-Fi).

O que é um servidor FTP?

Definição

Um servidor FTP permite transferir arquivos entre dispositivos usando o protocolo FTP, sendo usado para envio, download e gerenciamento remoto de arquivos.

Principais Características

  • Modos de operação: Ativo e Passivo.
  • Usado para hospedagem de sites, backups e compartilhamento de arquivos.
  • Softwares comuns: FileZilla Server, vsftpd.

Segurança

Por ser inseguro no formato tradicional, recomenda-se usar versões como:

  • FTPS: Com criptografia SSL/TLS.
  • SFTP: Baseado em SSH.

Principais funcionalidades do FTP em uma rede

1. Transferência de Arquivos

  • Envio (Upload): Envia arquivos para o servidor.
  • Recebimento (Download): Baixa arquivos do servidor.

2. Gerenciamento Remoto

Criar, renomear, excluir e listar arquivos e pastas no servidor.

3. Acesso Controlado

Autenticação por nome de usuário e senha, com permissões de acesso.

4. Modos de Transferência

  • ASCII: Para arquivos de texto.
  • Binário: Para arquivos não textuais (imagens, vídeos, etc.).

5. Transferência de Grandes Arquivos

Suporte para mover arquivos maiores que limites de e-mails.

6. Conexões Ativas e Passivas

Adaptável a diferentes configurações de rede e firewalls.

7. Multiplataforma

Funciona em diversos sistemas operacionais e redes, garantindo flexibilidade.

FTP: vantagens e limitações

Vantagens do FTP

  • Facilidade de Uso: O FTP é fácil de configurar e usar, com uma interface simples para usuários novatos e experientes.
  • Compatibilidade: É amplamente suportado em diversos sistemas operacionais, tornando-o versátil para diferentes plataformas.
  • Recursos Avançados: Permite transferências paralelas e a retomada de transferências interrompidas, aumentando a flexibilidade.

Limitações do FTP

  • Segurança: Transmite dados em texto claro, o que facilita a interceptação de informações, incluindo senhas e arquivos.
  • Desempenho em Redes Lentas: Pode ser ineficiente em redes com baixa largura de banda, causando lentidão nas transferências.
  • Falta de Criptografia: Não oferece criptografia nativa, o que o torna inadequado para transferências de dados sensíveis.

Diferenças entre FTP e Armazenamento em Nuvem

FTP

O FTP (File Transfer Protocol) é um protocolo usado para transferir arquivos entre computadores através de um servidor local. Para usá-lo, é necessário configurar um servidor, o que exige um certo conhecimento técnico. As transferências de arquivos no FTP precisam ser feitas manualmente e, sempre que há uma alteração, o arquivo precisa ser enviado novamente.

Armazenamento na Nuvem

O Armazenamento em nuvem permite que arquivos sejam armazenados em servidores remotos, acessíveis via internet. Ao contrário do FTP, a nuvem não exige que você gerencie servidores. Ela oferece maior facilidade de uso, com integração com dispositivos móveis, e recursos como sincronização automática de arquivos e controle de versões. A segurança na nuvem é frequentemente mais robusta, com criptografia e backups automáticos.

Servidor de e-mail e suas funções

O servidor de e-mail é essencial para enviar, receber e armazenar mensagens eletrônicas de forma eficiente e segura.

Funções principais

  • Envio de e-mails: Transmite mensagens utilizando o protocolo SMTP.
  • Recebimento de e-mails: Armazena mensagens com protocolos como POP3 e IMAP.
  • Armazenamento: Mantém os e-mails no servidor até serem acessados.
  • Segurança: Protege comunicações com autenticação e criptografia.
  • Integração: Trabalha com ferramentas como CRM e marketing digital.

SMTP e como ele funciona

O SMTP (Simple Mail Transfer Protocol) é o protocolo padrão para envio de e-mails na internet. Ele realiza o roteamento e a entrega de mensagens.

Funcionamento

  • Conexão e Autenticação: O cliente conecta ao servidor SMTP e se autentica.
  • Envio da Mensagem: Comandos SMTP como MAIL FROM e RCPT TO identificam remetente e destinatário.
  • Roteamento: O servidor verifica o domínio e consulta registros DNS se necessário.
  • Entrega Final: A mensagem é entregue ao servidor do destinatário.

Portas Comuns

  • Porta 25: Antiga padrão, mas atualmente bloqueada em muitos casos.
  • Porta 587: Padrão moderno com suporte à criptografia TLS.
  • Porta 465: Usada para conexões com SSL (menos comum).

Uso

O SMTP é usado para envio de e-mails, enquanto protocolos como IMAP ou POP3 cuidam da recuperação de mensagens.

POP3 e IMAP

POP3 (Post Office Protocol 3)

O POP3 baixa os e-mails do servidor para o dispositivo local e geralmente os remove do servidor após o download. É ideal para usuários que acessam e-mails em um único dispositivo.

Vantagens do POP3:

  • Armazenamento local dos e-mails.
  • Menor uso de espaço no servidor.
  • Acesso offline garantido.

Desvantagens do POP3:

  • Sem sincronização entre dispositivos.
  • Maior risco de perda de dados.

IMAP (Internet Message Access Protocol)

O IMAP mantém os e-mails no servidor, permitindo acessá-los de diferentes dispositivos e sincronizando as alterações em tempo real.

Vantagens do IMAP:

  • Sincronização em múltiplos dispositivos.
  • Melhor organização e recuperação de dados.
  • Menor risco de perda de e-mails.

Desvantagens do IMAP:

  • Requer mais espaço no servidor.
  • Maior dependência de conexão à internet.

Portas 25, 110 e 143

As portas são pontos de comunicação lógica usados pelos dispositivos para identificar e direcionar os dados que chegam ou saem. Cada porta é identificada por um número e está associada a um protocolo específico, permitindo que vários serviços funcionem simultaneamente em um mesmo dispositivo.

Porta 25

Usada pelo protocolo SMTP (Simple Mail Transfer Protocol), é a porta padrão para o envio de e-mails. Devido a problemas com spam, muitos provedores bloqueiam esta porta, recomendando alternativas como a porta 587 para conexões seguras.

Porta 110

Associada ao protocolo POP3 (Post Office Protocol 3), permite o recebimento de e-mails. Esta porta é utilizada para conexões não criptografadas, onde os e-mails são baixados do servidor para o cliente de e-mail e, em geral, removidos do servidor.

Porta 143

Utilizada pelo protocolo IMAP (Internet Message Access Protocol), permite acessar e-mails diretamente no servidor e sincronizar entre dispositivos. Assim como a porta 110, a porta 143 é usada para conexões sem criptografia.
