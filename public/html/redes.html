Fundamentos de Redes

A página de Redes é dedicada ao estudo e à aplicação de tecnologias que conectam dispositivos, permitindo a troca de informações de forma eficiente e segura. Aqui você encontrará conteúdos sobre a configuração de equipamentos de rede, como switches e roteadores, os principais protocolos utilizados (DNS, HTTP, FTP, SMTP, entre outros), e conceitos fundamentais como o modelo OSI e a pilha TCP/IP

Dispositivos para um Rede Interna

Uma rede interna típica com um servidor web geralmente inclui:

Roteador

Interliga diferentes redes e direciona pacotes de dados entre elas, garantindo comunicação externa.

Switches

Conecta dispositivos em uma rede local, permitindo a troca de dados entre eles.

Servidor

Dispositivo que oferece serviços, como páginas web, resolução de nomes ou distribuição de IPs, para outros dispositivos na rede.

Host(PC ou Laptop)

Computadores ou dispositivos que acessam os serviços disponíveis na rede.

Cabo de Conexão

Proporcionam a conexão física entre os dispositivos para transmissão de dados.

Configurações dos Servidores

Nas Configurações do Servidor da Rede Interna, o mais importante é:

DNS

Configurações incluem entrada de registros para mapear nomes de domínio a endereços IP.

Exemplo:

sa-bryan.netlify.app/
HTTP

Necessário especificar o diretório raiz onde os arquivos HTML e outros conteúdos estão armazenados.

DHCP

Configuração de um escopo com intervalo de endereços IP, máscara de sub-rede, gateway padrão e DNS.

Conceito de Servidor

Um servidor é um dispositivo ou software que fornece serviços ou recursos para outros dispositivos (clientes) em uma rede.

Vídeo da Crimpagem da Empresa

Para realizar a crimpagem, utilizamos um alicate de crimpagem específico e conectores RJ45 de alta qualidade. Cortamos o cabo de rede no tamanho desejado, organizamos os fios na sequência correta que é o padrão T568A e fixamos os conectores com precisão. Garantimos conexões seguras e funcionais para um excelente desempenho da rede.

Cabo de Rede - Pinagem

Pinagem padrão de um cabo de rede (RJ45) segue:

Padrão T568A

Verde/branco, verde, laranja/branco, azul, azul/branco, laranja, marrom/branco, marrom.

Padrão T568B

Laranja/branco, laranja, verde/branco, azul, azul/branco, verde, marrom/branco, marrom.

Conceito do modelo OSI

O modelo OSI (Open Systems Interconnection) é um modelo em 7 camadas que padroniza a comunicação em redes.

Conceito do TCP/IP

TCP/IP é uma pilha de protocolos de comunicação usados na internet. Ele tem 4 camadas.

O que é um servidor FTP?

Um servidor FTP (File Transfer Protocol) é um servidor usado para transferir arquivos entre clientes e servidores através de uma rede.

Principais funcionalidades do FTP em uma rede

Transferência de arquivos entre dispositivos. Armazenamento centralizado. Controle de acesso por autenticação.

FTP: vantagens e limitações

Vantagens: Simplicidade e eficiência em transferências de arquivos. Limitações: Falta de criptografia nativa, vulnerável a ataques.

Diferenças entre FTP e armazenamento em nuvem

FTP

Requer servidores próprios e é limitado em interface e colaboração.

Nuvem

Acessível pela web, integrada com ferramentas modernas, e oferece segurança aprimorada.

Servidor de e-mail e suas funções

Servidor de e-mail gerencia o envio, recebimento e armazenamento de mensagens eletrônicas. Exemplos: SMTP para envio e POP3/IMAP para recebimento.

SMTP e como ele funciona

SMTP (Simple Mail Transfer Protocol) é usado para envio de e-mails entre servidores ou de clientes para servidores. Usa a porta 25 (ou 587 para segurança).

POP3 e IMAP

POP3

Baixa e-mails do servidor para o cliente, removendo-os.

IMAP

Sincroniza e-mails entre o servidor e o cliente, mantendo os e-mails no servidor.

Portas 25, 110 e 143

As portas são pontos de comunicação lógica usados pelos dispositivos para identificar e direcionar os dados que chegam ou saem. Cada porta é identificada por um número e está associada a um protocolo específico, permitindo que vários serviços funcionem simultaneamente em um mesmo dispositivo.

Porta 25

Usada pelo SMTP para envio de e-mails.

Porta 110

Usada pelo POP3 para baixar e-mails do servidor para o cliente.

Porta 143

Usada pelo IMAP para sincronizar e-mails entre servidor e cliente.
